## Корневая поломка воркфлоу: контракт тела browserless
Документ (раздел 3/14.3): для кода browserless — ТОЛЬКО сырой JS
(`body: "=export default async function..."`, Raw, без JSON-обёртки).

- РАБОЧИЕ узлы (Пульс·снять, Куки·проверка/пинг, Разведчик·песочница):
  `sendBody:true`, тело = сырой JS `body: "=export default async function ({ page, context }) {...}"`.
  `context` (cookie/positions) берётся прямо из `{{ $json.cookie }}` внутри кода.
- СЛОМАННЫЕ «Руки» (Сканер·сбор, Sifariş·пуш, Вербовщик·отправка):
  `specifyBody: json`, `jsonBody: ={{ JSON.stringify({ code: `...`, context: { cookie: $json.cookie } }) }}`.
  Это JSON-конверт с полем `code` — не тот контракт, что ждёт сырой browserless-эндпоинт.
  → узел «не работает вообще».  (Мозг·рассылка шлёт form-параметры — отдельный косяк.)
- Куки-агент реализован ВЕРНО (раздел 10.10): читает `/home/node/.n8n/milli/cookie.json`,
  не зашит мёртвый. Но сам файл сейчас пустой/мёртвый — пока владелец не пришлёт
  свежий токен (F12→Console в кабинете), все browserless-ветки вернут `COOKIE_DEAD`.
  Это НЕ поломка кода, а ожидаемое состояние.
